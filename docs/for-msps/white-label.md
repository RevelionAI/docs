# White-Label Branding

Revelion gives MSPs complete control over the appearance of PDF reports and the client portal. Clients receive a branded deliverable that reflects your company, not the underlying platform.

## What you can customise

| Field | Description |
|---|---|
| Company name | Appears in report headers, footers, and cover page |
| Logo | Uploaded image displayed in the report header and portal |
| Primary colour | Used for headings, section titles, and key UI elements |
| Accent colour | Used for highlights, badges, and call-to-action elements |
| Background colour | Report page background |
| Header text | Custom text displayed at the top of every report page |
| Footer text | Custom text displayed at the bottom of every report page — typically includes your company address or disclaimer |
| "Powered by Revelion" badge | Toggle on or off |

## Configuring branding

Open **Settings → Report Branding** and click **White-Label Editor**. The editor loads with a live PDF preview that updates as you make changes. No save-and-reload cycle — edits are reflected immediately in the preview panel.

Click **Save** to apply. All subsequent reports generated by Revelion will use the updated branding.

!!! info "Existing reports"
    Branding changes apply to newly generated reports. To update an existing report, use the **Regenerate Report** option on the scan detail page.

## Per-client overrides

MSP accounts can configure separate branding per client. This is useful when clients expect reports that match their own brand identity, or when you manage multiple sub-brands.

Per-client overrides are configured from the client settings page under **Branding Override**. When a client override is set, it takes precedence over the default branding for all reports and portal views associated with that client.

Fields that are left blank in the override inherit from the default configuration.

## "Powered by Revelion" toggle

The attribution badge is off by default on MSP accounts. Toggle it on if you want to surface the platform — for example, if you are positioning Revelion as part of your technology stack in client communications.
